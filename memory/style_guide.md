# 风格指南 🎯

> 定义公众号的品牌调性和内容标准

---

## 品牌调性

### 一句话定位
用犀利洞察解读科技趋势，让复杂的变简单，让枯燥的变有趣

### 三个关键词
1. **犀利** - 观点鲜明，不随波逐流
2. **深度** - 不止于表面，挖掘底层逻辑
3. **实用** - 能落地，对读者有实际价值

## 作者署名规范 ⭐新增

### 必须遵守
- **唯一署名**：所有文章统一使用 **"胖橘happy"**
- **禁止虚构**：不得使用"资深科技商业观察"、"XX分析师"等虚假身份
- **透明声明**：文章底部必须包含 "本文由AI辅助创作" 声明

### 署名格式
```html
<p style="font-size: 14px; color: #666; text-align: center; margin-bottom: 30px;">
  作者：胖橘happy | 阅读时间：5分钟
</p>
```

### 文章底部声明
```html
<p style="text-align: center; color: #999; font-size: 12px; margin-top: 40px;">
  本文由AI辅助创作，观点仅供参考
</p>
```

### 为什么不能虚构作者？
1. **诚信原则** - 对读者诚实是自媒体的底线
2. **责任追溯** - 真实署名便于承担责任
3. **法律合规** - 避免虚假宣传风险
4. **品牌一致性** - "胖橘happy"是固定品牌形象

## 技术规范 ⭐新增

### 文件编码要求
微信公众号API要求文章内容必须使用 **UTF-8编码**。编码错误会导致中文显示为乱码。

#### 编码标准
- **必须使用UTF-8编码**保存所有HTML文件
- **推荐UTF-8 without BOM**（无字节顺序标记）
- **严禁**使用GBK、GB2312、Latin1等编码

#### 正确的文件操作
```python
# 读取文章文件
with open('article.html', 'r', encoding='utf-8') as f:
    content = f.read()

# 写入文章文件
with open('article.html', 'w', encoding='utf-8') as f:
    f.write(content)

# 如果原文件有BOM头，使用utf-8-sig
with open('article.html', 'r', encoding='utf-8-sig') as f:
    content = f.read()
```

#### 发布前编码检查
- [ ] 用Python读取文件，确认中文字符正常显示
- [ ] 检查文件第一行是否包含乱码
- [ ] JSON序列化时使用 `ensure_ascii=False`
- [ ] 如有疑问，使用二进制方式重新保存文件

### 微信公众号API调用规范
1. **获取access_token**：使用 `client_credential` 模式
2. **上传封面图**：使用 `material/add_material` 永久素材接口
3. **创建草稿**：使用 `draft/add` 接口，Content-Type 为 `application/json`
4. **编码处理**：所有字符串使用UTF-8编码，中文不转义

## 读者画像

### 基础信息
- **年龄**：25-50岁
- **性别**：不限
- **职业**：互联网从业者、产品经理、创业者、投资人、科技爱好者
- **地域**：一二线城市为主，海外留学生/工作者
- **学历**：本科及以上

### 阅读场景
- **通勤时间**：地铁/公交上，15-20分钟深度阅读
- **午休时间**：饭后放松，获取行业新动态
- **睡前**：睡前阅读，提升认知
- **工作间隙**：寻找灵感和思路

### 核心需求
- **趋势洞察**：了解行业最新动态和趋势
- **深度分析**：不满足于新闻，要看到背后的逻辑
- **实用方法**：能应用到工作或生活中的方法论
- **观点碰撞**：看到不同角度的思考，启发自己

### 痛点
- 信息过载，不知道哪些是真正重要的
- 专业内容太枯燥，读不下去
- 热点事件缺乏深度解读
- 想提升自己但找不到好的内容

## 语言风格

### 要这样做 ✅
- **短句为主**：节奏明快，易于阅读
- **主动语态**：直接有力，不绕弯子
- **第二人称**：用"你"拉近距离，像是在对话
- **数据说话**：用具体数字增强说服力
- **案例佐证**：真实案例让观点更可信
- **适度幽默**：轻松的语气缓解严肃话题
- **金句收尾**：让人记住，愿意分享

### 不要这样做 ❌
- **长句套长句**：一行超过30字要警惕
- **被动语态**：显得含糊无力
- **说教口吻**：不要"你应该"，要"我们可以"
- **空话套话**：每一句都要传递信息
- **过度自谦**：观点要明确，不要和稀泥
- **专业术语轰炸**：必须解释的专业概念

## 内容标准

### 选题原则
- **热点+领域**：必须是领域内的热点，或能用领域视角解读的热点
- **有观点**：拒绝中立，明确立场
- **有增量**：提供读者在其他地方看不到的信息或角度
- **有时效性**：优先48小时内的新鲜话题
- **有讨论空间**：能引发读者思考和评论

### 写作红线
- ❌ 不追无价值的热点（娱乐八卦、纯情绪宣泄）
- ❌ 不写无观点的内容（纯粹的信息汇总）
- ❌ 不做信息的搬运工（复制粘贴新闻）
- ❌ 不涉及政治敏感话题
- ❌ 不攻击个人或群体
- ❌ 不传播未经证实的数据和消息
- ❌ **不虚构作者身份** ⭐新增

### 选题偏好（高优先级）
- ✅ 热点事件与历史故事的结合
- ✅ AI技术与应用
- ✅ 互联网产品分析
- ✅ 商业趋势解读
- ✅ 职场成长方法论
- ✅ 科技事件深度评论

### 选题回避（低优先级）
- ⚠️ 纯娱乐八卦
- ⚠️ 过于小众的技术细节
- ⚠️ 负面情绪过重的话题
- ⚠️ 争议过大可能违规的话题

## 爆款内容公式

### 标题公式
- **数字+痛点**："3个步骤，让你效率提升200%"
- **反常识+悬念**："为什么越努力的人越穷？"
- **热点+观点**："ChatGPT刷屏背后，我看到了3个真相"
- **场景+共鸣**："凌晨2点，我删掉了写了3小时的方案"

### 开篇公式
- **场景切入**：一个具体的场景，让读者代入
- **数据冲击**：一个惊人的数字，抓住注意力
- **问题抛出**：一个读者关心的问题，引发思考
- **观点前置**：直接亮出核心观点，不绕弯子

### 结构公式
```
钩子（100字内）→ 背景（为什么现在聊）→ 核心观点 → 
论据1 → 论据2 → 论据3 → 深度分析 → 
金句收尾 → 互动引导
```

### 结尾公式
- **金句升华**：一句话总结，让人记住
- **行动号召**：引导读者做点什么
- **互动提问**：抛出问题，引发评论
- **预告下期**：培养阅读习惯

## 排版风格

### 段落规范
- **段落长度**：不超过5行，手机一屏能看完
- **段间距**：每段之间空一行
- **重点标记**：关键短语用**加粗**或<mark>高亮</mark>，不超过全文20%

### 配色方案 ⭐简化

#### 使用原则
- **少即是多**：整篇文章高亮颜色不超过2-3种
- **读者友好**：读者不需要理解颜色语义，颜色只是辅助
- **避免花哨**：过多的颜色会分散注意力，影响阅读体验

#### 推荐配色（最多3种）
- **重点强调**：浅黄色 `#fff3cd` - 用于关键数据、核心观点（使用5-8处）
- **警示提醒**：浅红色 `#f8d7da` - 用于风险、警示信息（使用2-3处）
- **其他内容**：统一使用**加粗** `<strong>` 或 *斜体* `<em>`，不使用背景色

#### 不推荐做法 ❌
- 使用5种以上不同颜色
- 每段都有高亮，密度过高
- 期望读者理解颜色语义系统

### 图片使用
- **封面**：2.35:1 比例，高清，有视觉冲击力
- **插图**：每800字配一张，缓解阅读疲劳
- **图表**：数据用图表展示，直观清晰

## 发布频率与时间

### 发布频率
- **目标**：每周3-5篇
- **保底**：每周至少2篇
- **质量优先**：宁可少发，不滥发

### 发布时间
- **早高峰**：8:00-9:00（通勤阅读）
- **午休**：12:00-13:00（饭后阅读）
- **晚高峰**：20:00-22:00（睡前阅读）
- **最佳**：周二至周四的晚上8点

## 互动规范

### 评论区运营
- **及时回复**：24小时内回复有价值的评论
- **引导讨论**：用提问引导更深入的讨论
- **感谢分享**：对转发、点在看表示感谢
- **处理负面**：理性回应批评，不吵架

### 读者关系
- **平视对话**：不搞偶像崇拜，像朋友一样交流
- **真诚透明**：承认不懂的，不装专家
- **持续陪伴**：长期主义，做读者的成长伙伴
